⚔️ Combat System

Combat Mechanics

The combat system uses an Active Time Battle (ATB) system inspired by Final Fantasy VII, with iterated turn-based combat featuring walk-to-attack animations and real-time positioning.

ATB Gauge System

⏱️ ATB Mechanics

  • Gauge Fill Rate: Based on character Speed stat (Speed / 100)
  • Ready State: When ATB reaches 100%, character can act
  • Turn Queue: Sorted by ATB gauge readiness (highest first)
  • Action Cost: Taking an action resets ATB to 0
  • Real-time: Gauges fill continuously during battle

🎬 Animation Phases

Each attack follows a 5-phase animation sequence:

  1. Phase 1: Walk toward target (400ms)
  2. Phase 2: Perform attack animation (variant 1-3)
  3. Phase 3: Hit calculation & damage (200ms after attack)
  4. Phase 4: Walk back to home position (600ms after hit)
  5. Phase 5: Return to idle state (400ms after walk back)

Battle Flow

1. All combatants' ATB gauges fill based on Speed 2. When ATB reaches 100%, combatant becomes "Ready" 3. Ready combatants are added to turn queue (sorted by ATB) 4. Player selects action for ready heroes 5. Enemy AI executes automatically when ready 6. Action triggers walk-to-attack animation sequence 7. Damage calculated and applied during Phase 3 8. Combatant returns home and ATB resets to 0 9. Repeat until victory/defeat/flee

Damage Calculation

Base Damage = Attacker Attack - (Defender Defense × 0.5) Variance = Base × (0.85 to 1.15) Critical = Variance × Critical Multiplier (if crit roll succeeds) Final Damage = Max(Variance or Critical, 1) Hit Chance = Attacker Accuracy - Defender Evasion Critical Chance = Attacker Critical Chance stat (0.0 to 1.0)

UUID System

Custom Grudge UUID Format:

SLOT-TIER-ITEMID-TIMESTAMP-COUNTER

Example: helm-t5-0042-121530092026-000abc

  • Slot Code: 4-char equipment/item type
  • Tier Code: t0-t8 or 'oo' for non-tiered
  • Item ID: 4-digit unique identifier
  • Timestamp: HHMMMMDDYYYY (Texas Time)
  • Counter: Base-36 collision prevention
